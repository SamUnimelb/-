本系统为温州肯恩大学体育场馆信息系统的一个设计方案，项目需求详见2022届大一计算机系期末项目文档。

该文档的视频解读见：https://www.bilibili.com/video/BV1fq4y1k7QS

本系统的正常运行依赖于以下工具和第三方包:
- python version 3.9.7
- MySQL server 5.6.51
- python第三方包: mysql和re

python的mysql链接器可以通过以下指令安装:
- pip install mysql
- pip install mysql-connector-python-rf

mysql链接参数在DBLinker.py中进行修改。

系统启动MySQL服务和python IDE后，运行ClassProcessor.py启动本系统。

按照终端提示即可使用本系统。

# 已知问题：

1. 本项目的数据库模型中含有对器材使用时间冲突检测的数据表，且项目中写了检查时间先后的函数，但限于时间和精力未实际写器材时间冲突管理功能，欢迎写了的小伙伴们pull requests补充呀！
2. 由于建模过程中以客户名直接作为主键，故客户重名时需要重新设计对应模型和该表。
3. 项目系适用python所写，仅可满足几百几千人的规模较小的大学健身场馆日常管理使用。若学校规模较大或场馆器材更丰富，建议使用C++或Java等更适合高并发后台的语言进行重构。

# 待提高项

1. 当前客户端仅有场馆管理员可用，后期建议项目甲方可考虑提出升级客户端为GUI且适用于多平台客户和教练的多系统终端使用需求。
2. 建议甲方可考虑提出存储教练训练过程中产生的图文和视频信息，这一点在数据库对应位置存储LONGBLOB文件即可。
